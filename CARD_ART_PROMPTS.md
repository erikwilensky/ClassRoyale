## A) MASTER PROMPT TEMPLATE

```
Clean vector icon style with subtle painterly shading, depicting {CARD_CONCEPT}. Bold silhouette, high contrast, minimal detail. {PALETTE} color palette with {CATEGORY_MOTIF}. Centered subject with 15% safe margin, minimal background. No text, no letters, no watermark, no humans, no faces, no hands. 1:1 aspect ratio, 1024x1024 resolution.
```

**Placeholders:**
- `{CARD_NAME}` - Full card name (e.g., "Brainwave Boost")
- `{CARD_CONCEPT}` - One-sentence visual concept
- `{CATEGORY}` - Category name (e.g., "TIME/TEMPO")
- `{PALETTE}` - Color palette description
- `{CATEGORY_MOTIF}` - Category-specific visual motifs

## G) DALL-E GENERATION PARAMETERS

### Recommended Settings

**Size:** `1024x1024` (1:1 aspect ratio)

**Quality:** 
- `standard` for initial generation (faster, lower cost)
- `hd` for final production images (higher detail, better consistency)

**Style:** 
- `natural` (recommended for consistency across all cards)
- Alternative: `vivid` if you want more vibrant colors (may reduce consistency)

**Number of Variations:** 
- `n: 1` for initial generation
- `n: 2-3` for cards that need refinement or multiple options

### Seed Strategy

Use fixed seeds per category to maintain visual consistency within each category:

- **TIME/TEMPO:** `seed: 101`
- **COMMS/SUGGESTIONS:** `seed: 202`
- **DECK/CASTING:** `seed: 303`
- **DEFENSE/COUNTER:** `seed: 404`
- **GOLD/ECONOMY:** `seed: 505`
- **DISRUPTION:** `seed: 606`
- **COSMETIC:** `seed: 707`

**Note:** DALL-E 3 does not support explicit seed parameters in the API. Instead, use the style anchor images as reference. Generate the 7 style anchors first, then use them as visual references when generating cards in each category.

### Generation Workflow

1. **Phase 1: Style Anchors**
   - Generate all 7 style anchor images first
   - Review for consistency and adjust prompts if needed
   - These serve as visual references for the category style

2. **Phase 2: Category Batches**
   - Generate cards by category (e.g., all TIME/TEMPO cards together)
   - Use the same quality setting for all cards in a batch
   - Review each batch before moving to the next

3. **Phase 3: Refinement**
   - Regenerate any cards that don't match the style
   - Use `n: 2-3` to get variations for problematic cards
   - Compare against style anchors for consistency

## I) NOTES & RECOMMENDATIONS

1. **Background Choice:** Since you selected "either" for background preference, prompts are written to work with both transparent and gradient backgrounds. DALL-E 3 will generate with backgrounds by default. You can remove backgrounds in post-processing if needed.

2. **Consistency:** Generate all cards in a category together to maintain visual consistency. The style anchors help establish the baseline.

3. **Iteration:** If a card doesn't match the style, regenerate with slight prompt adjustments rather than changing the entire template.

4. **Post-Processing:** Consider using image editing tools to:
   - Remove backgrounds if transparent PNGs are needed
   - Adjust colors slightly to match category palettes exactly
   - Ensure all images are exactly 1024x1024

5. **Testing:** Generate 1-2 cards from each category first to validate the prompts before generating all 50.
